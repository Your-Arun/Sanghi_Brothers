Technical Handover Documentation

This interactive manual is designed for the technical engineering team taking over the maintenance and deployment of the Sanghi Brothers petrol station suite.

The Sanghi Brothers Petrol Pump Management System is a full-stack, enterprise-grade management portal designed to streamline day-to-day operations at fuel stations. It automates nozzle readings, shift logs, cashier sheets, employee attendance tracking, accounts ledgers (Lekhajokha), sales, and scheduling. It also features automatic SMS reports to staff using Twilio services.

Key Features Summary

Attendance & Shifts
Tracks employee attendance details (Present, WFH, Leave, Absent) and maps staff shift duty rosters automatically with live updates.
Daily Pump Sheets
Automates daily nozzle sales logging, density checks, opening/closing meter indices, and generates instant reports.
Financial Ledger
Lekhajokha double-entry bookkeeping ledger, bank monthly inflows/outflows records, SB01 and SB03 transaction trackers.
SMS Broadcasts
Integrates node-cron with Twilio SMS api to broadcast formatted nozzle rosters to employees' mobile phones in real-time.

System Architecture

Data flow and integration design

The system is built on a modular Client-Server paradigm. The Frontend acts as a Single Page Application (SPA) communicating with the Backend REST API Server asynchronously using standard HTTP verbs via Axios.

Data Models

Database Schemas (Mongoose ODM)

All collections are managed via Mongoose schemas inside the Backend/models/ directory.

User Schema

models/user.js

Stores registered profiles containing username, email, phone, hashed password, and authorization departments. Access levels are strictly checked against the department field.
Attendance Schema

models/attendancewala/Attendance.js

Stores user check-in data with unique compound keys. Fields: userId, date, and status (Present, Absent, Leave, Holiday, WFH).
Shift Snapshot

models/shifting/MapSnapshot.js

Contains specific nozzle assignments (N1 to N6), supervisor details, Air boy, and active shift rosters for any target date. Used by the SMS Broadcast Bot.
Lekha (Bookkeeping)

models/lekha.js

Transactions logs for petrol pump accounts bookkeeping. Keeps record of item dates, narration details, vouchers, and credit/debit balances.

SMS Scheduler Bot

Automated Twilio Shift Roster Notifications

The scheduler script at Backend/models/shifting/smsBot.js coordinates the auto-dispatch of pump duty lists to workers daily via SMS.

Scheduling Logic

  • Dynamic Cron Settings: restartScheduler() queries the Settings model inside MongoDB to fetch customized shift times (default 05:00 and 14:00) and constructs the cron schedules dynamically.
  • Timezone Alignment: Cron threads operate strictly inside the Asia/Kolkata timezone.
  • Trigger Hooks: The bot is instantiated inside server.js upon startup. Any updates to scheduling configs trigger a hot restart of the bot jobs.

Production Deployment

Steps for Vercel and VPS/Cloud hosting

Frontend Hosting (Vercel)

The frontend React application is optimized for Vercel deployment. It contains a vercel.json file at the root to rewrite SPA route paths cleanly, preventing router breakdowns when pages are reloaded.

vercel.json 
{
  "rewrites": [
    { "source": "/(.*)", "destination": "/" }
  ]
}

Backend API Hosting

To deploy the server to production platforms (such as Render, Railway, or VPS instances):

  • Set the target deployment directory context to Backend/.
  • Setup the build / startup trigger hook: npm run start (calls node server.js).
  • Add all the keys listed in the Environment Settings tab to the hosting provider environment dashboard panel.
  • Set the client CORS domains lists inside Backend/server.js to accept requests from your production web app URL.
